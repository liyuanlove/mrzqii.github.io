JavaScript设计模式与开发实践1-基础

发表于 2018-05-15 | 分类于 JS

已经不止一次被推荐的书了，前段时间刚刚把JavaScript高级程序看了一遍，想把里面的知识再次的融合，于是经过朋友推荐，开始完成这本书。

每天忙完工作的事情已经比较晚了，只能慢慢的完成这本书了，我会把我觉得重要的一些思想和一些知识点进行总结。便于以后查阅。

开始：

设计模式其实就是解决某一类问题对应的方法，其实平时我们写代码的时候或多或少已经使用了设计模式，只是自己不知道而已；

使用设计模式可能会给增加代码量和程序的复杂度，软件开发的成本并非全部在开发阶段，还在可复用性，可维护性方面。好的设计模式就能很好的进行后期维护和代码复用

第一章

编程语言分类
1：静态语言类型
->在编译时便已确定变量的类型
2：动态语言类型 ->要到程序运行的时
候，待变量被赋予某个值之后，才会具有某种类型。

鸭子类型:通俗说法是：“如果它走起
路来像鸭子，叫起来也是鸭子，那么它就是鸭子。”
鸭子类型指导我们只关注对象的行为，而不关注对象本身
多态：其实是对象的多态性；当我们对一些函数发出“调用”的消息时，这些函数会返回不同的执行结果，这是“多态性”的一种体现

多态的思想实际上是把“做什么”和“谁去做”分离开来，要实现
这一点，归根结底先要消除类型之间的耦合关系。

封装：其他对象或者用户都不关心它的内部实现。封装使得对象之间的耦合变松散，对象之间只通过暴露的 API 接口来通信
把系统中稳定不变的部分和容易变化的部分隔离开来
通过原型模式实现继承：将对象构造器的原型指向另外一个对象，继
承总是发生在对象和对象之间。

第二章

this的指向大致分为此四类：
作为对象的方法调用。
作为普通函数调用。
构造器调用。
Function.prototype.call或Functin.prototype.apply调用。

  1. 当函数作为对象的方法被调用，this指向这个对象
  2. 当函数不作为对象的属性被调用时，也就是我们常说的普通函数方式，此时的 this 总是指向全局对象。在浏览器的 JavaScript 里，这个全局对象是 window对象。
  3. 当用 new运算符调用函数时，该函数总会返回一个对象，通常情况下，构造器里的 this就指向返回的这个对象，但用 new调用构造器时，还要注意一个问题，如果构造器显式地返回了一个 object类型的对
    象，那么此次运算结果最终会返回这个对象，而不是我们之前期待的 this
  4. Function.prototype.call或 Function.prototype.apply可以动态地
    改变传入函数的 this
    apply和call:
    apply 接受两个参数，第一个参数指定了函数体内 this 对象的指向，第二个参数为一个带下
    标的集合，这个集合可以为数组，也可以为类数组，apply 方法把这个集合中的元素作为参数传
    递给被调用的函数
    call:传入的参数数量不固定，跟 apply相同的是，第一个参数也是代表函数体内的 this指向，
    从第二个参数开始往后，每个参数被依次传入函数
    当调用一个函数时，JavaScript 的解释器并不会计较形参和实参在数量、类型以及顺序上的
    区别，JavaScript 的参数在内部就是用一个数组来表示的。从这个意义上说，apply比 call的使用
    率更高，我们不必关心具体有多少参数被传入函数，只要用 apply一股脑地推过去就可以了。
    有时候我们使用 call 或者 apply 的目的不在于指定 this 指向，而是另有用途，比如借用其

第三章

闭包

谈到闭包就离不开
变量的作用域以及变量的生存周期
有var声明的变量是局部变量
没有var申明的变量是全局变量，准确的说是window对象上面的一个属性
上面两点的区别：

  1. 作用域不同
  2. 一个会声明提前
  3. var声明的不能用delete删除。而作为对象的属性是可以删除的

全局变量的生存周期是永久的，除非我们主动销毁这个全局变量。
而对于在函数内用 var关键字声明的局部变量来说，当退出函数时，这些局部变量即失去了它们的价值，它们都会随着函数调用的结束而被销毁，但是有一种情况就是在闭包发生的时候 局部变量所在的环境还能被外界访问，这个局部变量就有了不被销毁的理由。在这里产生了一个闭包结构，局部变量的生命看起来被延续了

闭包在开发中常见的作用：

  1. 封装变量
  2. 延续局部变量的寿命

闭包与内存管理机制：

  1. 有人说闭包会造成本来应该在程序结束就销毁的变量一直不被销毁。其实我们要思考为什么我们要使用变量，可能是因为在以后还需要这个变量，把这些变量放在闭包中和放在全局作用域，对内存方面的影响是一致的，这里并
    不能说成是内存泄露
  2. 跟闭包和内存泄露有关系的地方是，使用闭包的同时比较容易形成循环引用如果两个对象之间形成了循环引用，那么这两个对象都无法被回收，但循环引用
    造成的内存泄露在本质上也不是闭包造成的。
